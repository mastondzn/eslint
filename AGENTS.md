# AGENTS.md

## Package manager

pnpm only (pinned in `packageManager` field). Use `pnpm install` / `pnpm run <script>`.
CI uses `@antfu/ni` shorthands (`nr` = `pnpm run`, `nci` = `pnpm install --frozen-lockfile`).

## Commands

```sh
pnpm run typegen       # regenerate src/typegen.d.ts (required before build/typecheck)
pnpm run build         # typegen + tsdown --clean --dts → dist/
pnpm run stub          # tsdown only, no DTS, no clean (fast local stub)
pnpm run watch         # tsdown --watch
pnpm run lint          # eslint .
pnpm run format        # prettier --write .
pnpm run format:check  # prettier --check . (CI-safe)
pnpm run typecheck     # tsc --noEmit
pnpm run dev           # @eslint/config-inspector (interactive rule browser)
pnpm run release       # bumpp (version bump) + pnpm publish
```

## Required ordering

1. After a fresh clone, run `pnpm run typegen` before `pnpm run typecheck` — `src/typegen.d.ts` is gitignored and must be generated locally.
2. Full CI sequence: **build → lint → format:check → typecheck**.
3. `pnpm run build` internally runs `typegen` first, then `tsdown --clean --dts`.
4. `prepack` runs `build` automatically before publish — no manual step needed.

## No tests

There is no test suite. No jest, vitest, or other test runner is configured. Correctness is validated by linting and typechecking the source itself.

## Generated file

`src/typegen.d.ts` (~17 k lines) is generated by `scripts/typegen.ts` via `eslint-typegen`. It is gitignored. Typecheck will fail without it. Regenerate with `pnpm run typegen` whenever rules or plugins change.

## Linting without a build

`eslint.config.ts` imports from `./src` (not `dist/`). ESLint v10 loads the TS config via `jiti`, so `pnpm run lint` works without a prior `pnpm run build`. The `dist/` directory is also gitignored and not committed.

## Plugin prefix renames

This config renames several plugin prefixes. Use these when writing inline `// eslint-disable` comments in this repo's own source:

| Use this prefix | Original prefix        | Plugin                             |
| --------------- | ---------------------- | ---------------------------------- |
| `ts/*`          | `@typescript-eslint/*` | `@typescript-eslint`               |
| `import/*`      | `import-x/*`           | `eslint-plugin-import-x`           |
| `node/*`        | `n/*`                  | `eslint-plugin-n`                  |
| `yaml/*`        | `yml/*`                | `eslint-plugin-yml`                |
| `test/*`        | `vitest/*`             | `@vitest/eslint-plugin`            |
| `react/*`       | `@eslint-react/*`      | `@eslint-react/eslint-plugin`      |
| `tailwindcss/*` | `better-tailwindcss/*` | `eslint-plugin-better-tailwindcss` |

## Release flow

- `pnpm run release` → `bumpp` (interactive version bump) + `pnpm publish`.
- Pushing a `v*` tag triggers the release workflow, which runs `changelogithub` to create GitHub release notes from commits.
